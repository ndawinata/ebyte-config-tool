Remote configuration (Fixed-point)

For remote configuration, the local module must use fixed-point transmission and share the same NET ID as the remote node. Many desktop RF tools forward GET/SET using a proprietary 0xCF 0xCF prefix to the target address.

Note: public documentation for remote frames is incomplete for all variants. This tab sends an experimental CF CF C1 … sequence through the open port for testing via the local module.

Download / Firmware mode

Desktop “Download” mode is often used to push firmware to a companion MCU (e.g. E22-DTU). Use this tab to send raw bytes for link testing.

Multi-device UART terminal

Choose how many USB-serial terminals you need, then create the panels. Each slot has its own baud rate, RX log, and send line. Put each E22 in transparent mode (M0=0, M1=0) for over-the-air UART pass-through. This tab is independent from the configuration toolbar above.

You can recreate panels anytime; open ports in this tab will be closed first.

No panels yet. Set the number of devices and click Create panels.

How to use

  1. Open this app in Chrome, Edge, or Opera (Web Serial API).
  2. Set the E22 module to configuration mode: M0=1, M1=1.
  3. Select the Band / channel map preset (or Custom MHz), Profil label daya if your RF Setting shows 22/17/13/10 dBm instead of 30/27/24/21, then Open Port and choose your USB-Serial device.
  4. Click Get Param to read the active register map into the form.
  5. Edit fields as needed, then Set Param to write permanently to flash.
  6. When finished, return jumpers to normal mode (M0=0, M1=0) before using the radio on-air.

Protocol frames

  • C1 00 09 — Get Param (9 register)
  • C0 00 09 [9 byte data] — Set Param (permanent)
  • C2 00 09 [9 byte data] — Set Param (volatile)
  • C1 80 07 — Get Product Information (raw PID bytes; toolbar only — not ASCII model name)

Multi-device terminal

Use the Multi-device terminal tab: set the device count, click Create panels, then connect each USB‑serial adapter in its slot. Match baud to the module’s UART. Use Send as hex for raw frames; otherwise text is UTF‑8 (optional LF).
